# Contributing

First off, thank you for considering contributing to this project! Every contribution, whether it's fixing bugs, improving documentation, or adding new features, is appreciated.

## Before You Start

**Please do not open a Pull Request immediately.**

Before working on any changes, you **must**:

1. Join the project's Support Server.
2. Contact **`seafarers_identity_document`**.
3. Discuss the feature, fix, or improvement you'd like to work on.

Once approved, you'll be added to the **Developer** group, where you can coordinate with other contributors, avoid duplicate work, discuss implementation details, and get help if needed.

## Development Process

After being added to the Developer group:

* Coordinate with other developers before starting major work.
* Keep discussions and planning within the developer channels.
* If you're unsure about an implementation, ask before spending time on it.
* Feel free to suggest improvements or new ideas.

## Pull Requests

When you're ready to submit your work:

* Make sure your changes have been discussed beforehand.
* Keep Pull Requests focused on a single feature or fix.
* Write clear commit messages.
* Describe what your Pull Request changes and why.
* Be open to feedback and requested changes during review.

## Code Quality

Please ensure that your contributions:

* Follow the existing coding style and project structure.
* Keep the code clean, readable, and maintainable.
* Avoid unnecessary dependencies unless discussed beforehand.
* Test your changes before submitting.

## Questions

If you have any questions about contributing, development, or the project in general, join the Support Server and contact **`seafarers_identity_document`**.

We look forward to working with you. Happy coding! 🚀
